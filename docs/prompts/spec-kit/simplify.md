# Simplify spec, plan, or design

```text
Simplify {{TARGET_FILE}}.

Do not remove research-critical requirements.
Do not implement code.
Do not run provider-backed commands.

Prefer the simplest design that preserves:
- research validity;
- reproducibility;
- traceability;
- artifact contracts;
- metric provenance;
- phase separation;
- migration expectations.

Remove or defer speculative abstractions, excessive metadata, unnecessary layers, broad rewrites,
or future-proofing not justified by current specs and known domains.

Report:
- what was simplified;
- what was preserved and why;
- follow-up changes needed.
```
